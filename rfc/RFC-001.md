# RFC-001: Database Selection

Version: 0.1
Status: Draft
Date: 2026-07-15

---

# 배경

SPEC-310은 Persistence를 추후 결정으로 두고 있다. 현재 repository 인터페이스만 고정되어 있고, 실제 저장소가 없어 이벤트 소싱과 감사 로그 정책을 구현할 수 없다.

---

# 제안

PostgreSQL로 MVP persistence를 진행한다.

## 이유

- 관계형 모델이 Aggregate 단위 저장과 Soft Delete 규격과 일치한다.
- JSONB로 가변 metadata를 저장할 수 있다.
- AI 결과/외부 데이터 참조 저장에 적합하다.

## 대안

- SQLite: 개발 편의성은 좋으나 운영 환경에서 scaling에 제한이 있다.
- MongoDB: 스키마 유연성은 좋으나, 관계형 규칙과 트랜잭션 경계가 복잡해진다.
- Event Store 전용: 의존성과 운영 비용이 크다.

---

# 계획

- SPEC-400 Database에 PostgreSQL/Migration/Repository 구현 계약을 추가한다.
- 앱은 repository 인터페이스만 보고 DB 교체 가능 구조를 유지한다.
