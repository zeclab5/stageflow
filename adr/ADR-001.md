# ADR-001 : Specification First

Status: Accepted
Date: 2026-07-15

---

# 결정

모든 구현은 Specification 먼저 작성 후 진행한다. Specification 없이는 코드를 작성하지 않는다.

---

# 배경

프로젝트의 일관성과 품질 유지가 필요하다. 구현 우선은 아키텍처 붕괴와 코드 품질 저하로 이어진다.

---

# 결정 이유

- 설계와 구현의 용이한 추적.
- 복수 AI 참여시 컨텍스트 공유.
- 리팩토링 및 유지보수 비용 절감.

---

# 영향

- 모든 작업 전 SPEC 작성이 필수.
- SPEC 리뷰 없이 구현 Merge 불가.
