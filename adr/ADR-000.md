# ADR-000 : Architecture Freeze 0.1

Status: Accepted
Date: 2026-07-15

---

# 결정

StageFlow 아키텍처를 Hexagonal Architecture + DDD + Event Driven + Plugin Architecture로 Freeze한다.

---

# 배경

공연 제작 워크플로우는 외부 시스템 연동이 잦고, 비즈니스 규칙이 계속 변한다. 이에 대응 가능한 아키텍처가 필요했다.

---

# 결정 이유

- Hexagonal Architecture : 외부 시스템과의 경계를 명확히 분리.
- DDD : 복잡한 공연 제작 도메인을 명확히 모델링.
- Event Driven : 상태 변경을 이벤트로 표현하여 확장성 확보.
- Plugin Architecture : Resolume 등 외부 프로그램과의 연동을 플러그인으로 격리.

---

# 영향

- 모든 모듈은 경계를 존중해야 한다.
- 직접 DB 호출, 외부 시스템 직접 연동은 금지.
- 변경은 RFC를 통해서만 가능.
